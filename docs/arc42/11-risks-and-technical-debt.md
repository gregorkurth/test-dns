# arc42 Kapitel 11: Risiken und technische Schulden

## Zweck

Hier stehen Dinge, die uns Sorgen machen oder spaeter Arbeit kosten koennen.

## Was hier hinein gehoert

- bekannte Risiken
- offene Architekturfragen
- Luecken in Tests oder Nachweisen
- Medienbruch-Risiken beim Export
- technische Schulden

## Wann pflegen?

- wenn neue Risiken erkannt werden
- wenn Risiken kleiner werden
- wenn wir bewusst Schulden akzeptieren

## Quelle im Repo

- Features mit offenen Punkten
- ADRs
- QA-Ergebnisse
- Release-Review
