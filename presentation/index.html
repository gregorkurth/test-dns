03Single Source of Truth

Warum SSOT?

📍

Traceability

  • Jedes Requirement hat eine eindeutige ID (SREQ-xxx)
  • Jeder Test verlinkt auf sein Requirement
  • Jedes Feature referenziert die zugehörigen Requirements
🔄

Automatisierung

  • Skills aktualisieren INDEX.md automatisch
  • Tech Design wird direkt in OBJ-X.md geschrieben
  • QA-Ergebnisse fliessen in die Feature-Spec ein
🔒

Git als Wahrheit

  • Kein Word, kein Confluence, kein Jira
  • Alle Änderungen sind nachvollziehbar
  • Airgapped-fähig: funktioniert ohne Cloud
💡
Neue Information gehört immer zuerst in eine Datei in diesem Repo. Nicht in E-Mails, nicht in Chats, nicht in Confluence. Wenn es nicht im Repo ist, existiert es nicht.
