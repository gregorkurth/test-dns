# Service: Release Management

> **Service ID:** SVC-DEV-RELEASE
> **Capability:** DevOps (CAP-005)
> **Quelldokument:** App-Template-Anweisung, Abschnitt 10 und 10a

---

## Beschreibung

Release Management stellt sicher, dass Releases einer SemVer-Logik folgen, der CHANGELOG automatisch generiert wird, Release-Artefakte signiert und nachvollziehbar sind und ein Publish erst nach erfolgreicher Prüfung des finalen Artefaktinhalts erfolgt.

---

## Service Functions

| ID | Service Function | Beschreibung |
|----|-----------------|-------------|
| SFN-DEV-003 | Semantic Versioning | SemVer-Tags und automatischer CHANGELOG |
| SFN-DEV-004 | Release Automation | GitHub Releases mit signierten Artefakten |
