## Anforderungstext (Original)

> For any zones that are not anycasted, responses from the authoritative name servers must contain the same source IP address as the destination IP address of the initial query.

## Anforderungstext (Deutsch)

Für Zonen, die nicht über Anycast betrieben werden, müssen Antworten der autoritativen Nameserver dieselbe Source-IP-Adresse enthalten wie die Destination-IP-Adresse der ursprünglichen Anfrage. Die Antwort muss von derselben IP-Adresse kommen, an die die Anfrage gerichtet war.

---

## Kontext

Diese Anforderung stellt sicher, dass DNS-Antworten von der erwarteten Adresse stammen. Asymmetrisches Routing (Anfrage an IP A, Antwort von IP B) kann zu Problemen bei Firewalls und ACLs führen. Im Anycast-Betrieb ist dieses Verhalten technisch bedingt anders, weshalb die Ausnahme explizit genannt wird.

---

## Akzeptanzkriterien

1. DNS-Antworten für Nicht-Anycast-Zonen haben als Source-IP dieselbe Adresse wie die Destination-IP der Anfrage.
2. BIND9 ist so konfiguriert, dass es auf der Adresse antwortet, auf der die Anfrage empfangen wurde.
3. Netzwerk-Capture zeigt Source-IP = Original-Destination-IP für Nicht-Anycast-Queries.
